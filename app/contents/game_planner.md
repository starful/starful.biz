# [完全ガイド] Game Planner: ゲームプランナーの年収・将来性は？未経験からのロードマップ

## 導入：Game Plannerという職業の「光と影」

「ゲームが好きだから、ゲームを作る仕事がしたい」
そんな純粋な動機でこの門を叩く若者は後を絶ちません。しかし、断言しましょう。**ゲームプランナーという仕事は、あなたが想像しているよりも100倍は泥臭く、1000倍は胃が痛くなる仕事です。**

世間一般のイメージでは、ゲームプランナーはお洒落なオフィスでコーヒーを片手に「こんな面白いアイデアを思いついた！」とホワイトボードに絵を描いている、クリエイティブな王様のように見えるかもしれません。しかし、現実はまるで違います。

実際の現場でのプランナーは、**「仕様の矛盾をエンジニアに詰められ、デザイナーには工数の無茶振りをなじられ、プロデューサーからは売上目標の未達を叱責される、チームの防波堤」**です。

朝、意気揚々と出社しても、昨晩リリースしたイベントの不具合でユーザーからSNSで叩かれ、緊急メンテナンスの対応に追われる。昼食を摂る暇もなく、1セルでも間違えれば数千万円の損失が出る「パラメータ設計」のExcelシートと格闘する。そして深夜、誰もいなくなったオフィスで、「面白さとは何か」という正解のない問いに自問自答しながら、数百ページの仕様書を書き上げる……。

それでも、なぜこの職種がIT・エンタメ業界でこれほどまでに熱望されるのか。それは、**「自分の妄想が、数百万人の感情を動かし、熱狂を生み出す瞬間の快感」**が、他のどんな職種でも味わえない中毒性を持っているからです。

この記事では、現役のエキスパートとして、ゲームプランナーという職業の「残酷な現実」と、その先にある「至高のやりがい」を、忖度なしのフルボリュームで解説します。覚悟を持って読み進めてください。

## 💰 リアルな年収相場と、壁を越えるための「残酷な条件」

ゲームプランナーの年収は、二極化が激しいのが特徴です。単なる「作業員」で終わるか、プロジェクトを牽引する「軍師」になれるかで、生涯賃金には億単位の差がつきます。

| キャリア段階 | 経験年数 | 推定年収 (万円) | 年収の壁を突破するための「リアルな必須条件」 |
| :--- | :--- | :--- | :--- |
| ジュニア | 1-3年 | 300 - 450 | 言われたことをこなすだけでなく、仕様の「なぜ」を理解し、デバッグやデータ入力の正確性で信頼を勝ち取れるか |
| ミドル | 3-7年 | 450 - 750 | チームのボトルネックを特定し、エンジニアやデザイナーの工数を最適化しながら、KPI（売上・継続率）に直結する施策を立案・完遂できるか |
| シニア/リード | 7年以上 | 750 - 1,200+ | 経営層と技術の橋渡しを行い、数億〜数十億円規模の予算責任を負い、プロジェクトの「死」を回避する意思決定ができるか |

### なぜ、あなたの年収は頭打ちになるのか？
多くのプランナーが年収500万円前後で停滞します。その理由は明確です。**「面白いことを考える」だけで、それを「形にするための調整」と「稼ぐためのロジック」を放棄しているからです。**

シニアクラスが1,000万円を超えるのは、彼らが「クリエイター」である以上に「ビジネスマン」であり「プロフェッショナルの調整役」だからです。
例えば、あるプロジェクトで「新機能を追加したい」という要望が出たとき、ジュニアは「面白そうですね！」と言います。ミドルは「工数はどれくらいですか？」と聞きます。しかし、**シニアは「その機能を追加することでDAU（1日の利用者数）が何％向上し、開発費を何ヶ月で回収できるか。そして、既存のコードベースにどれほどの負債を残すか」を即座に計算し、時には「やらない」という決断を下します。** この「NO」と言える責任の重さが、年収の差となって現れるのです。

## ⏰ Game Plannerの「生々しい1日」のスケジュール

華やかなゲーム画面の裏側にある、血の通った（あるいは血の気の引く）1日の流れを見てみましょう。これは、ある中規模スマホゲーム運用のリードプランナー、Aさん（32歳）の例です。

*   **09:30：出社・不穏な朝会**
    オフィスに到着するなり、Slackの通知が止まらない。昨晩20時に開始した期間限定イベントの「ドロップ率」に設定ミスが発覚。SNSでは「絞りすぎ」「詐欺だ」と炎上気味。朝会ではエンジニアから「この仕様、そもそも考慮漏れがあったんじゃないか？」と冷ややかな視線を浴びる。謝罪しつつ、修正案をその場で提示。
*   **11:00：仕様レビュー（通称：戦場）**
    来月実装予定の新キャラのスキル仕様書をエンジニアとデザイナーに説明。
    「このエフェクト、今のメモリ制限じゃ無理ですよ」「このスキル、対人戦のバランス崩壊しませんか？」
    次々と飛んでくるツッコミ。プランナーはここで「でも面白いから」という精神論に逃げてはいけない。代替案を提示し、技術的な妥協点を探りつつ、本来の面白さを死守する。
*   **13:00：ランチ（という名の情報収集）**
    競合他社の新作ゲームをプレイしながら、コンビニのサンドイッチを流し込む。これも仕事だが、心休まる暇はない。「なぜこのUIは使いやすいのか？」「この課金導線はエグいな」と分析が止まらない。
*   **14:00：データ分析・パラメータ調整**
    午後は集中タイム。SQLを叩いて昨日のユーザー行動を抽出。
    「レベル15で離脱者が急増している。このステージの難易度が高すぎるのか？」
    Excelの数式と格闘しながら、敵のHPを0.1%単位で調整する。この地味な作業が、ゲームの寿命を左右する。
*   **16:00：プロデューサーとの密談**
    「来月の売上予測、あと1,500万上乗せできないか？」という無茶振りが降ってくる。
    既存の施策を削るか、新たなキャンペーンをねじ込むか。開発工数とチームの疲弊具合を天秤にかけながら、胃を痛めて交渉する。
*   **18:30：本番障害発生**
    退勤準備を始めた頃、サーバー監視アラートが鳴り響く。特定条件下でアプリが強制終了するバグが判明。
    「原因特定まで帰れませんね」というエンジニアの言葉に、夜食の買い出しを覚悟する。プランナーの役割は、影響範囲の特定と、ユーザーへの告知文作成、そして現場の士気を下げないための振る舞いだ。
*   **21:00：孤独な仕様作成**
    障害対応が落ち着き、ようやく自分の作業。静まり返ったオフィスで、再来月のイベントプロットを書く。
    「この物語で、ユーザーは泣いてくれるだろうか？」
    昼間の殺伐とした調整とは打って変わり、純粋に「面白さ」と向き合うこの時間だけが、プランナーの魂を救ってくれる。
*   **23:00：退勤**
    深夜のタクシーの中で、自社のゲームの掲示板を見る。「今回のイベント、神運営だな」という一言を見つけ、少しだけ救われた気持ちで眠りにつく。

## ⚖️ この仕事の「天国（やりがい）」と「地獄（きつい現実）」

### 【やりがい：天国】
1.  **「世界」を創り、数百万人に届ける快感**
    自分が考えたキャラクター、自分が設計したバトルシステムが、世界中の人々にプレイされる。電車の中で隣の人が自分のゲームを遊んでいるのを見かけた時の、あの「心の中でガッツポーズする感覚」は、何物にも代えがたい。
2.  **数字という「残酷で正直な結果」との対峙**
    施策を打てば、翌日には数字として結果が出る。仮説が的中し、売上グラフが垂直立ち上がりした時の全能感。データ分析に基づき、狙い通りにユーザーを動かせた瞬間、あなたは「神」に近い視点を得ることになります。
3.  **多才なプロフェッショナルとの共創**
    自分一人では絶対に作れないクオリティの絵、音、プログラム。それらが自分の仕様書を核として集結し、一つの作品として結実していくプロセスは、オーケストラの指揮者のような高揚感があります。

### 【きつい部分：泥臭い現実】
1.  **「面白さ」という正解のない問いに殺される**
    どんなにロジカルに説明しても、「で、これ面白いの？」という一言で全てがひっくり返ることがあります。自分の感性を否定され続け、何が正しいのか分からなくなる夜が必ず来ます。
2.  **全方位からの「板挟み」と「サンドバッグ」**
    ユーザーからは「無能運営」と叩かれ、社内からは「仕様が遅い」「工数見積もりが甘い」と責められる。責任はプランナーにあるが、権限はプロデューサーにある……という歪な構造の中で、メンタルを削りながら調整を続ける必要があります。
3.  **積み上げた努力が「一瞬で消える」無常観**
    数年かけて開発したタイトルが、リリースからわずか数ヶ月でサービス終了（サ終）が決まる。その時の喪失感は、自分の子供を失うような痛みです。IT業界の中でも、これほどまでに「成果物が残らない」虚しさを味わう職種は他にありません。

## 🛠️ 現場で戦うための「ガチ」スキルマップと必須ツール

ゲームプランナーに求められるのは「アイデア」ではありません。そのアイデアを「実現可能な形に翻訳する能力」です。

| スキル・ツール名 | 現場での使われ方（「なぜ」必要なのか、具体的なシーン） |
| :--- | :--- |
| **Excel / Google Sheets** | 聖典。複雑な関数やマクロを駆使し、ゲームバランスのシミュレーションや膨大なアイテムデータの管理を行うため。 |
| **SQL** | ユーザー行動ログを自ら抽出するため。分析チームを待たずに「今、何が起きているか」をデータで証明できないプランナーは不要。 |
| **ロジカルシンキング** | 「なぜこの機能が必要か」をエンジニアに納得させるため。感情論ではプロは動かない。論理の穴を突かれたら仕様は死ぬ。 |
| **UXデザイン / ワイヤーフレーム** | 画面遷移図やUI案を作成するため。デザイナーに「いい感じに」と頼むのは三流。意図を視覚的に伝える必要がある。 |
| **Unity / Unreal Engine (基礎)** | 開発環境で直接数値をいじったり、簡単なプロトタイプを作るため。エンジニアの作業を止めずに微調整を行う自走力が求められる。 |
| **交渉力・心理学** | チームの士気を高め、無理なスケジュールを通すため。相手の「やりたくない理由」を先回りして潰す泥臭いネゴシエーション。 |

> **【プロの視点】**
> 多くの初心者が「企画力」を磨こうとしますが、現場で最も重宝されるのは「**仕様の完遂力**」です。どんなに素晴らしいアイデアも、仕様書の端々に矛盾があれば、開発現場は混乱し、最終的にクソゲーが出来上がります。1ピクセルのズレ、1フラグの矛盾を許さない「執念」こそが、最強のスキルです。

## 🎤 激戦必至！Game Plannerの「ガチ面接対策」と模範解答

ゲーム会社の面接官（特に現場のリード級）は、あなたの「ゲーム愛」など見ていません。彼らが見ているのは、**「この修羅場を一緒に乗り越えられる、論理的でタフなパートナーか」**という一点です。

### 質問1：「あなたが最近プレイしたゲームで、一つだけ改善点を挙げてください」
*   **面接官の意図**: 分析的思考力と、ユーザー視点・開発視点の両立を確認したい。単なる「不満」ではなく、ビジネス的なトレードオフを理解しているか。
*   **NGな回答例**: 「ガチャの確率が低いので、もっと上げるべきだと思います。そうすればユーザーが喜ぶからです。」（←ビジネスを無視した子供の意見）
*   **評価される方向性**: 「〇〇というゲームのUIにおいて、強化画面への遷移が3タップ必要な点を1タップに改善すべきです。なぜなら、現在の仕様は育成のテンポを阻害しており、データ上もそこで離脱が起きていると推測されるからです。ただし、ショートカット実装による画面の煩雑化というリスクについては、〇〇という形で回避します。」

### 質問2：「エンジニアから『その仕様は技術的に不可能だ』と言われたらどうしますか？」
*   **面接官の意図**: コミュニケーション能力と、問題解決への執着心を見たい。
*   **NGな回答例**: 「頑張って説得します」または「諦めて別の案を考えます」。
*   **評価される方向性**: 「まず、なぜ不可能なのかという技術的な制約（CPU負荷、メモリ、工数など）を詳細にヒアリングします。その上で、私が実現したい『面白さの本質』がどこにあるかを再定義し、その本質を損なわずに技術的制約をクリアできる代替案を、エンジニアと一緒に模索します。」

### 質問3：「自分が面白いと確信している企画が、チーム全員から反対されました。どうしますか？」
*   **面接官の意図**: 客観性と、エゴのコントロールができているか。
*   **NGな回答例**: 「自分の感性を信じて押し通します」または「みんなが言うならボツにします」。
*   **評価される方向性**: 「反対の理由を言語化してもらいます。もしそれが『主観的な好み』であれば、プロトタイプを作ってテストプレイをするか、類似タイトルの成功データを提示して客観的な判断材料を揃えます。逆に、反対理由が『工数対効果』などの妥当なものであれば、速やかに執着を捨て、次の案に切り替えます。目的は私のエゴを通すことではなく、ゲームを成功させることだからです。」

### 質問4：「KPI（売上など）が目標に届かなかった時、あなたはどう責任を取りますか？」
*   **面接官の意図**: 責任感の所在と、PDCAを回せるかを確認したい。
*   **NGな回答例**: 「次は頑張ります」「運が悪かったと思います」。
*   **評価される方向性**: 「数字が出なかった原因を多角的に分析します（流入経路、継続率、課金転換率など）。その分析結果をチームに共有し、失敗を『知見』に変えた上で、最短でリカバリーするための修正施策を即座に提案・実行します。プランナーの責任の取り方は、辞めることではなく、数字を立て直すことです。」

### 質問5：「ゲームプランナーにとって、最も必要な素養は何だと思いますか？」
*   **面接官の意図**: 仕事の本質をどう捉えているか。
*   **NGな回答例**: 「想像力です」「ゲームを愛する心です」。
*   **評価される方向性**: 「『サービスを完遂させる執念』と『客観性』です。どれほど高い理想を掲げても、リリースできなければ価値はゼロです。また、自分のこだわりがユーザーの利益に反していないかを常に疑う客観性がなければ、独りよがりの作品になってしまうからです。」

## 💡 未経験・ジュニアからよくある質問（FAQ）

### Q1. プログラミングスクールを出ただけでなれますか？
**A. 正直に言いましょう。なれません。**
プログラミングスキルはプラスですが、プランナーに求められるのは「コードを書くこと」ではなく「何を作るかを定義すること」です。スクールで学んだことよりも、**「自分でゲームを企画し、Unity等で一本完成させ、ストアにリリースして、ユーザーのフィードバックを受けた経験」**の方が100倍評価されます。未経験なら、能書きよりも「動く実績」を持ってきてください。

### Q2. 数学の知識はどこまで必要ですか？
**A. 確率統計と中学・高校レベルの数学は「必須」です。**
ガチャの確率計算、ダメージ計算式の設計、経済圏のインフレシミュレーション……。これらは全て数学です。「文系だから数字は苦手」というプランナーは、現場では「計算できない会計士」と同じくらい無力です。特にExcelでシミュレーションモデルを組める能力は、あなたの生存率を劇的に高めます。

### Q3. 30代未経験からでも挑戦できますか？
**A. 極めて厳しいですが、不可能ではありません。**
ただし、20代の若者と同じ土俵で戦うのは無理です。前職での「プロジェクトマネジメント経験」「深いデータ分析スキル」「特定の業界知識（競馬、歴史、ミリタリー等）」など、**ゲーム制作以外の武器をプランナーの業務にどう転用できるか**を証明する必要があります。即戦力としての「大人の振る舞い」が期待されます。

### Q4. アイデアノートをたくさん作っています。評価されますか？
**A. ほとんど評価されません。**
「アイデア」そのものに価値はありません。業界には星の数ほどのアイデアが転がっています。評価されるのは、そのアイデアを「どうやって期間内に、この予算で、このターゲットに向けて実装し、どうやって利益を出すか」という**実行プラン**です。ノートを見せるなら、収支計画と仕様書セットで見せてください。

### Q5. ゲームプランナーの将来性は？AIに奪われませんか？
**A. 「作業」は奪われますが、「意思決定」は残ります。**
仕様書の清書やバランス調整の補助、素材生成などはAIがやるようになるでしょう。しかし、「今の時代の空気に、どの面白さが刺さるのか」「チームの士気をどう高めるか」「理不尽なトラブルにどう落とし所をつけるか」という、**極めて人間臭い意思決定と調整**はAIには不可能です。泥臭い人間関係の中で価値を発揮できるプランナーの需要は、むしろ高まっていくでしょう。
